WSDL (Web Services Description Language) — это язык описания веб-сервисов, основанный на XML. На нем описываются методы, входные и результирующие структуры данных, типы данных, сетевые адреса для обращения к сервису и другое.

ASMX предоставляет возможность создавать веб-службы, которые отправляют сообщения с помощью протокола SOAP.

ASMX - это старый способ создания веб-сервисов, а WCF - текущий способ создания веб-сервисов. Вся разработка новых веб-сервисов SOAP на клиенте или сервере должна выполняться с использованием WCF.
Можно сказать, что сервисы ASMX являются частным случаем WCF-сервисов, НО эта технология довольно проста и удобна в использовании, поэтому Microsoft ее оставил отдельно.

Сетевые сервисы – это службы, которые объединяют пользователей в группы и позволяют им совместно заниматься различными видами деятельности.

«переходник» — Service Proxy – набор настроек, описывающих сервис для клиентского приложения.

-1) веб-сервисы - это основанная на XML платформенно-независимая технология разработки распределенных приложений
-2) веб-сервисы — это реализация абсолютно четких интерфейсов обмена данными между различными приложениями, которые написаны не только на разных языках, но и распределены на разных узлах сети
-3) Web - сервисы — это программы, доступ к которым осуществляется по протоколу HTTP, а обмен данными происходит в формате XML.
4) веб-сервис — это приложение предоставляющее открытый интерфейс, пригодный для использования другими приложениями в Web

***IIS (Internet Information Services, до версии 5.1 — Internet Information Server) — проприетарный набор серверов для нескольких служб Интернета от компании Майкрософт. IIS распространяется с операционными системами семейства Windows NT.
Основным компонентом IIS является веб-сервер, который позволяет размещать в Интернете сайты. IIS поддерживает протоколы HTTP, HTTPS, FTP, POP3, SMTP, NNTP.

CLI (Common Language Infrastructure) – спецификация общеязыковой инфраструктуры. Определяет архитектуру исполнительной системы и набор представляемых сервисов. 
**Common Language Infrastructure ( CLI ) является открытой спецификацией (технический стандарт) , разработанный Microsoft и стандартизованный ISO и Ecma, которая описывает исполняемый код и среду выполнения, что позволяет несколько языков высокого уровня, которые будут использоваться на различных компьютерные платформы без переписывания для конкретных архитектур. Языки CLI - это языки компьютерного программирования, которые используются для создания библиотек и программ, соответствующих спецификациям Common Language Infrastructure (CLI). 

CLR (Common Language Runtime) – общеязыковая среда исполнения, виртуальная машина на которой исполняются  все приложения, работающие в среде .NET. Реализация CLI VES компанией Microsoft. Компилятор JIT(Just in Time). 
**Другими словами CLR этот тот самый механизм, который позволяет программе выполняться в нужном нам порядке, вызывая функции, управляя данными. И все это для разных языков (c#, VisualBasic, Fortran). Да, CLR действительно управляет процессом выполнения команд (машинного кода, если хотите) и решает, какой кусок кода (функцию) от куда взять и куда подставить прямо в момент работы программы.

MSIL(Microsoft IL) – реализация CLI CIL компанией Microsoft.
IL (Intermediate Language) — код на специальном языке, напоминающим ассемблер, но написанном для .NET. В него преобразуется код из других языков верхнего уровня (c#, VisualBasic). 

FCL (Framework Class Library) –  реализация CLI BCL компанией Microsoft. Можно рассматривать, как API CRL. 
.NET Framework – программная технология  программирования.  Это совокупность: CLR ( +JIT).
